In general, there are 2 ways to access Actyx logs. You can either use Actyx tools, or you can directly access them on your host.

:::info
You can only use the Actyx CLI and Node Manager to access logs once Actyx has successfully started. If you are investigating startup errors, you need to access logs directly from your host.
:::

<TabItem value="cli">


Use the `ax logs tail` command to get the logs of a node:

```text
ax logs tail --local 192.168.2.10
```

If you want to specify how many logs `ax logs tail` should return, or just want to follow logs as they are created, check our [reference documentation](../../reference/cli/logs/tail) for usage examples.

:::tip Use bash to automate collection of logs
If you are familiar with bash, you can use it to automate the collection of logs:

```bash
for ip in <IP address list>; do ax logs tail --all-entries --local $ip > $ip.log; done`
```

As you can see in the above snippet, this will collect **all log entries** and save them in one file per node.
:::

Check our [reference documentation](../../reference/cli/logs/tail) for usage examples.

</TabItem>

<TabItem value="linux">


On Linux and macOS, Actyx automatically outputs logs into your terminal after startup:

```text
./actyxos-mac
ActyxOS started.
Apr 21 10:33:35.882  INFO axosnode::logs_handler: Sending new dynamic config for logsvcd: DynamicConfig { node_id: SourceId(FIXME), node_name: "Default Node", monitoring_topic: Some("default-topic/monitoring") }
Apr 21 10:33:35.883  INFO logsvcd::sync: Received new dynamic config DynamicConfig { node_id: SourceId(FIXME), node_name: "Default Node", monitoring_topic: Some("default-topic/monitoring") }
more logs...
```

</TabItem>
<TabItem value="docker">


To see logs after starting Actyx on Docker in your terminal, you need to set the `ENABLE_DEBUG_LOGS` environment variable when running it. Add `-e ENABLE_DEBUG_LOGS=1` to your `docker run` command, and you will see logs appearing in your terminal.

In case the Actyx container is running in non-interactive mode the background, use `docker logs -f <container id>` to follow the logs as they are written.

</TabItem>
