When deploying Actyx in a production environment, the way you operate your solution is up to you. However, we strongly recommend setting up a Unified Endpoint Management (UEM) system.

:::info Conceptual guide on operations
For a holistic view and explanations on operating an Actyx solution, check out our [guide on operating Actyx production deployments](../../conceptual/operations.mdx).
:::

## Choosing the right tool

Depending on the use case, the kind of devices, and its operating system, the UEM you can use varies. Based on our experiences, we can recommend using one of the following solutions.

:::info
Please note that you should consider not only physical but also virtual devices, such as virtual machines. 
:::

### For mobile devices

For mobile devices, such as Android tablets, we can recommend using:

- [Workspace One UEM](https://www.vmware.com/products/workspace-one/unified-endpoint-management.html)
- [SOTI](https://soti.de/)

### For headless devices

For headless devices, such as machine gateways, we can recommend using:

- [Balena](https://www.balena.io/)

:::info
There are a lot of different UEM systems on the market, and the best solution for your use case might be different from the systems listed above.
:::
